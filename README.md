# bithatch-tfg-dam

El proyecto debe integrar backend, frontend, base de datos, seguridad, testing, despliegue y documentación.

Incluye el desarrollo de una __API REST__, documentada con __Swagger__, securizada con __JWT__ y con pruebas unitarias e integración. 

El __cliente móvil__, desarrollado en __React Native__, debe consumir la API, __funcionar parcialmente en modo offline__, tener arquitectura modular, __múltiples pantallas__, __autenticación__, gestión de perfil de usuario, modo claro/oscuro y una funcionalidad extra que use sensores del dispositivo. El ___cliente web administrativo___ es __opcional__.

La __interfaz__ debe cumplir criterios de __accesibilidad (WCAG)__, __usabilidad (Nielsen)__, diseño visual coherente y ser __responsive__.

Se _requiere_ la implantación de __Odoo__ como ERP, con justificación _técnica y funcional_.

El proyecto debe incluir un ___plan de sostenibilidad___ y un ___plan de marketing___.

Se deben implementar servicios __REST__ y __SOAP__, aplicando medidas de _seguridad_ como __JWT__.

El _despliegue_ contempla __servidor web configurado__ y uso de __certificados SSL__.

Finalmente, es ___obligatoria___ una __documentación completa__, con memoria del proyecto, __manuales de usuario__ y configuración, y un __vídeo demostrativo__ de la aplicación.
